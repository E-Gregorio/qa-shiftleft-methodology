1. Propósito

Este calendario detalla las fases, actividades y dependencias de las pruebas de todo el proyecto.

Es un documento vivo que guía la planificación y permite seguimiento preciso de avances, riesgos y retrasos.

4. Criterios de Seguimiento

  • Las actividades se monitorean semanalmente (o según el ritmo del sprint).
  • Toda desviación ≥10% del plan se reporta como riesgo.
  • La replanificación requiere aprobación del Líder QA y del Project Manager.

5. Integración con la Base de Datos

El calendario se relacionará con las tablas:

  • epics / user_stories: para alinear las fechas de prueba de cada historia.
  • test_suites / test_cases: para monitorear ejecución en tiempo real.
  • progress_reports: para generar métricas automáticas de cumplimiento.
